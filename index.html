FT.com Style Guide

using the FT's code library

The following pages show the layout, style, colours and dimension instructions for all components on the FT.com website. Adhere strictly to this style guide to create the correct uniformity and consistency across all the FT.com's site elements. If you have any queries please contact the FT.com Design & UX team.

GRID

Grab the grid file

This PSD contains a rough grid file you can to use while layout out your page. Grid file

Have a look at the non-article page examples to get some ideas on how you can layout your page.

The FT site width is 972px. All page components are designed to fit within the FT grid. For example the right-hand rail (RHR) is always 6 divisions wide. The grid consists of 16 divisions 42px wide with 20px gutter between each. When creating new module, widgets or components make sure your widths conform to the FT grid. Always use 20px gutter's between items.

MODULES

A module is a container that holds a piece of content. There are many different types of that exist in the Main Content Column (MCC) or in the Right-hand Rail (RHR) column. Content varies from promotional boxes, feeds to signposting to other content within the site.

All modules have one Velcro unit of space (30px) at the bottom.

Main Content Column Modules

There are many different types of element types that only appear in the MCC.

Page title (Primary Level)

These are used on non-article pages to give the page a name. Note: Do not use standard article titles to create page titles for non-article pages. This referred to as a primary level item.

Secondary module title (Secondary Level)

These appear most commonly on the homepage and different section home pages on the site (e.g. World, Companies, Markets, Global Economy etc). Section homepages are referred to as “FT index pages”.

On the Homepage for example the More Stories, Multimedia and Offers module use this style. See example below

Content bundle with title (Tertiary Level)

Content bundle untitled (Tertiary Level)

This is a list of content which wont necessarily have a title. There are already a few types in existence. The example below is called a Midriff and exists on section pages or the homepage.

Midriff

A midriff is an small area that interrupts a list of articles to add some variation to the page layout. Midriffs are used to surface other content often unrelated to what’s on the page (but not always). The example below holds a list of articles from the UK section in the first column the left (A) and a single page Special Report in the second column on the right (B)

Footer Rule

Velcro contains a footer style which is a simple line. This is used as a device to visually seperate page elements. However there are certain cases where the footer is not needed. Below are the rules for how footers whould be used

  1. Insert the Velcro footer under a module when it is followed by ordinary content (example in Midriff sample above).
  2. If a module is followed by another module do not use a footer.

Right-hand Rail Modules

RHR modules are stacked on top of each other on the right-hand side of the page. They are always 6 grid columns wide or 342px.

Examples of type of modules that exist in the RHR; RSS feed, Twitter feed, Most read article, Social media links, Other articles in this section, Latest stories from...
